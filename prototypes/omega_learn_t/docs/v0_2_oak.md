# Ω-LEARN-T v0.2 OAK note

## Distinctions

- A score is not proof of mastery.
- A scheduled review is not learning unless followed by active recall or transfer.
- A residue is not a failure; it is an error-correction asset.
- A GitHub issue is not a cognitive truth; it is an operational task.

## Required loop

```text
spec -> diagnose -> queue -> active test -> evidence -> M-/M+ -> OAK -> canon candidate
```

## Acceptance

A skill can become a canon candidate only after delayed recall, a new variant, transfer, explanation and captured residues.
