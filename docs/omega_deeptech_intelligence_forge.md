# Ω-DeepTech Intelligence Forge

**Status:** v0.1 MVP scaffold, OAK-safe.

Ω-DeepTech Intelligence Forge is the operational layer that turns Tristan's research ecosystem into a repeatable pipeline:

```text
papers + patents + startups + grants + repo signals
-> LOG/CVCD extraction
-> OAK risk and evidence gate
-> IP classification
-> prototype tasks
-> revenue candidates
-> GitHub artifacts
```

It is intentionally narrower than Zapier, Copilot, PatSnap, Elicit or general agent platforms. The niche is **Canada/Quebec deeptech intelligence + IP triage + prototype generation + revenue routing**.

## Core modules

1. **AgentFabric** — collects events from safe sources and converts them into normalized signals.
2. **SAGE Digest** — summarizes papers, patents, startups, funding and geopolitical signals.
3. **IPForge** — classifies ideas as `open_public`, `patent_review`, `trade_secret`, `service_revenue`, or `discard`.
4. **OAKGate** — blocks overclaims, missing sources, unsafe disclosure, legal/IP risks and unverifiable claims.
5. **RevenueForge** — maps validated signals to services, reports, licenses, tools, courses, audits, bounties or consulting.
6. **GitHubForge** — turns validated actions into issues, branches, docs, tests and PRs.

## MVP output

A weekly or daily intelligence packet:

```text
1. source signal
2. extracted claim
3. evidence level
4. OAK risks
5. IP class
6. prototype action
7. revenue action
8. GitHub next artifact
```

## OAK rules

- Do not confuse hypothesis, market signal, patentability, novelty and proof.
- Do not disclose potentially patentable inventions publicly before IP review.
- Do not send commercial email or external submissions without explicit approval.
- Always record sources, assumptions, uncertainty, counter-hypotheses and negative memory.
- Prefer dry-run and draft artifacts before irreversible actions.
