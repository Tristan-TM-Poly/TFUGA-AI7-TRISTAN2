# Ω-AIT-RESEARCH-FACTORY-T
## L’usine de recherche, prototypes, preuves et revenus de Tristan

> **Status:** canonical layer above CanonOS.  
> **Hard boundary:** ResearchFactory plans research, prototype, benchmark, IP/publication, and revenue experiments. It does not execute legal, financial, medical, public, or irreversible actions without the relevant review gates.

## 0. Idée mère

CanonOS gouverne le savoir vivant. RealityForge forge les idées. ResearchFactory transforme les branches fertiles en trajectoires productives.

```math
\boxed{AIT_{ResearchFactory}=CanonOS+RealityForge+ExperimentEngine+PrototypeLine+BenchmarkLine+IPGate+RevenueGate+PublicationGate}
```

But:

```math
\boxed{\text{transformer chaque branche fertile en preuve, prototype, actif, publication ou revenu}}
```

## 1. Canon -> Factory

CanonOS dit:

```text
cette idée est R/P/C niveau X
```

ResearchFactory demande:

```text
quelle ligne de production doit-elle prendre ?
```

Lignes possibles:

```text
Theory Line
Prototype Line
Benchmark Line
Paper Line
IP Line
Product Line
Internal Tool Line
Dataset Line
Education Line
Consulting Line
```

## 2. Experiment Engine

Toute hypothèse forte doit produire une expérience minimale:

```text
hypothesis
baseline
metric
test data
expected failure
success condition
residue
next iteration
```

L’idée ne reste pas seulement bonne. Elle devient expérimentable.

## 3. Prototype Line

```text
theory -> minimal API -> CLI -> tests -> benchmark -> demo -> README -> OAK report -> draft PR
```

Règle:

```math
\boxed{\text{pas de prototype sans test; pas de claim sans benchmark}}
```

## 4. Benchmark Line

```math
\text{valeur réelle}=\text{performance}-\text{baseline}-\text{coût}-\text{risque}-\text{complexité}
```

Niveaux:

```text
B0 none
B1 toy_example
B2 synthetic_test
B3 simple_baseline
B4 standard_baseline
B5 realistic_dataset
B6 stress_test
B7 reproducible_benchmark
B8 external_comparison
B9 controlled_use
B10 field_standard
```

## 5. IP / Publication Gate

Avant toute sortie publique, classer:

```text
open_source
private_internal
potential_ip
technical_note
scientific_publication
service_asset
license_candidate
```

Règle:

```math
\boxed{\text{ne jamais publier un actif sensible avant classification IP, privacy et OAK}}
```

## 6. Revenue Gate

Une idée peut devenir revenu si elle résout un problème réel.

```math
RevenueScore=Pain+Urgency+Budget+Repeatability+Proof-Risk-Competition-Complexity
```

Mais:

```math
\boxed{\text{revenu potentiel}\neq\text{preuve scientifique}}
```

## 7. Factory Scheduler

```text
si idée faible -> RealityAnchor
si hypothèse -> expérience
si prototype -> tests
si outil testé -> benchmark
si benchmark fort -> publication/IP/revenu review
si risque fort -> quarantine/review
si dette forte -> réduire dette
```

## 8. Proof-to-Asset Compiler

```text
proof/result -> technical note -> demo -> benchmark -> pitch -> IP classification -> product hypothesis -> partner list -> revenue experiment
```

## 9. M− Factory

```text
failed test -> M− -> better test -> better benchmark -> better prototype -> stronger canon
```

Un échec bien enregistré augmente la valeur future du système.
