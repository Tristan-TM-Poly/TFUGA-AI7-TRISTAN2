# Public Trust Stack — 8e Feu Site

Status: B — trust architecture
Date: 2026-07-06

## Purpose

The public site must not rely on charisma alone. It must build trust through visible structure.

```text
identity -> values -> proof -> code -> tests -> reports -> products -> contact
```

## Stack levels

### 1. Identity

```text
Who is Tristan?
What does he build?
Why should anyone care?
```

### 2. Values

```text
OAK, JKD, Gaia, 8e Feu, respect, proof discipline.
```

### 3. Proofs

```text
Merged PRs, tests, CLI, reports, bundles, public artifacts.
```

### 4. Code / artifacts

```text
GitHub repositories, packages, schemas, generated outputs.
```

### 5. Tests

```text
CI, pytest, validation gates, reproducible commands.
```

### 6. Reports

```text
Markdown reports, OAK reports, proof ledgers, product maps.
```

### 7. Products

```text
Audits, sprints, prototypes, GovTech demos, public intelligence systems.
```

### 8. Contact

```text
Minimal, direct, privacy-aware collaboration path.
```

## Product trust card template

Every product should show:

```yaml
name:
what_it_does:
what_it_does_not_do:
inputs:
outputs:
proof_status:
price_status:
risks:
next_action:
```

## Trust failure modes

```text
strong identity but no proof -> add proof ledger
strong proof but no offer -> add product card
strong offer but no limits -> add OAK disclaimer
strong symbolism but no respect gate -> add respect notice
strong tech but no contact -> add CTA
```

## Final rule

```text
Trust is not claimed. Trust is assembled.
```
