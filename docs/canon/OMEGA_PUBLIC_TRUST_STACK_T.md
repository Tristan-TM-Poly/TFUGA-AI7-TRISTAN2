# Ω-PUBLIC-TRUST-STACK-T

Status: B — public trust canon
Date: 2026-07-06

## 0. Role

Ω-PUBLIC-TRUST-STACK-T defines how the public site earns trust.

```text
identity -> values -> proof -> artifacts -> tests -> reports -> products -> contact
```

## 1. Core axiom

```text
Trust is not claimed. Trust is assembled.
```

## 2. Trust layers

1. Identity — who Tristan is and what he builds.
2. Values — OAK, JKD, Gaia, responsibility, respect.
3. Proof — merged PRs, tests, CLI, reports, generated artifacts.
4. Artifacts — code, schemas, bundles, graphs, public docs.
5. Tests — CI, pytest, gates, reproducibility.
6. Reports — OAK reports, proof ledger, product maps.
7. Products — audits, sprints, prototypes, company paths.
8. Contact — low-friction collaboration path.

## 3. Anti-overclaim rule

```text
If the claim becomes larger than the proof, shrink the claim or grow the proof.
```

## 4. Public labels

Use public OAK badges:

```text
Vision / Doctrine / Prototype / Tested / Merged / Product Candidate / Commercial Hypothesis / Validated Offer / Company Path
```

## 5. Result

The public site becomes a proof interface, not a vanity page.
