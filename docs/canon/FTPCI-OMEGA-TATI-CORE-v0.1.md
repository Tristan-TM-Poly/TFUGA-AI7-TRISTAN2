## 4. Factorisation propre

```math
\mathcal T
\approx
\mathcal G
\times_1 U_A
\times_2 U_D
\times_3 U_I
\times_4 U_E
\times_5 U_P
\times_6 U_O
\times_7 U_F
\times_8 U_M
```

- `G` = noyau compressé ;
- `U_*` = facteurs interprétables et traçables.

Règle JKD :

```math
Compute(\mathcal G) \quad avant \quad Compute(\mathcal T)
```

On ne calcule sur le tenseur dense que si le noyau compressé ne suffit pas.

## 5. Conditions de propreté

Une factorisation est propre si elle respecte :

1. **Traçabilité** : chaque facteur renvoie à des sources.
2. **Sparsité** : le noyau est beaucoup plus petit que le tenseur dense.
3. **Conservation d'invariants** : les invariants utiles survivent à la compression.
4. **Distance OAK bornée** : la reconstruction ne dépasse pas l'erreur admise.
5. **Fertilité positive** : le noyau produit des descendants utiles.
6. **Mémoire négative intégrée** : les anciennes erreurs réduisent les chemins futurs.
7. **Décompression sélective** : le système ne reconstruit que ce qui est utile.
8. **Canonisation rare** : tout élément promu possède statut, limites, test ou preuve.

Forme compacte :

```math
Clean(\mathcal G) = Traceable \land Sparse \land InvariantSafe \land OAKSafe \land Fertile
```

```math
\mathcal R_k = \mathcal T_k - \widehat{\mathcal T}_k
```

Le résidu contient :

- contradictions non résolues ;
- théories manquantes ;
- signaux faibles ;

Score de découverte :

```math
D(x)=Residual(x)\cdot Fertility(x)\cdot OAKPotential(x)
```

Priorité :

```math
x^* = \arg\max_x D(x)
```
