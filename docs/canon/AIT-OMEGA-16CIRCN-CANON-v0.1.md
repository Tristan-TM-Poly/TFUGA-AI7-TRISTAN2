# AIT-Ω 16^{○n} Canon v0.1

**Statut OAK :** architecture exécutable initiale, non autonome hors dépôt, non connectée à des API payantes.  
**Objet :** définir l'AIT Tristan comme organisme de transformation vérifiable : génération, attaque, test, mémoire, codex, prototype, GitHub.

---

## 1. Définition

```math
AIT_{\Omega}^{16^{\circ n}}
=
\left[
Top16
\circ
OAK
\circ
Score_{\Omega}
\circ
Generate16
\circ
Compress
\circ
Memory
\right]^{\circ n}
```

Le symbole `16^{○n}` ne signifie pas une énumération dense. Il signifie une composition récursive contrôlée : générer 16, attaquer, scorer, garder Top16, enterrer Bottom16, décompresser le fertile, puis recommencer.

---

## 2. Différence fondamentale

```text
chatbot -> répond
agent -> agit
swarm -> multiplie les agents
harness -> structure les workflows
AIT -> transforme, vérifie, mémorise, prototype, régénère
```

Un AIT supérieur n'est pas un modèle plus bavard. C'est une architecture qui laisse des traces vérifiables.

---

AIT_0=(Mission,Contract,Tools,M^+,M^-,OAK,Metrics,Limits,Prototype)

## 6. Score Ω

```math
Score_\Omega(x)=
\frac{
Fertility(x)\cdot OAK(x)\cdot Reuse(x)\cdot PrototypeValue(x)\cdot Impact(x)\cdot Transmission(x)
}{
Cost(x)+Risk(x)+Hype(x)+Complexity(x)+Debt(x)+1
}
```

Décision :

```math
Top16=\arg\max_{16} Score_\Omega
```

```math
Bottom16=\arg\min_{16} Score_\Omega
```

```math
Bottom16\rightarrow M^-
```
