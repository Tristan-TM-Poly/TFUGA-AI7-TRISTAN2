## 2. Tours LOG/EXP

### 2.1 Tour LOG

```math
LOG(X)=[L_0(X),L_1(X),L_2(X),L_3(X),L_4(X),L_\Omega(X)]
```

- `L0` = données brutes
- `L1` = motifs locaux
- `L2` = invariants
- `L3` = règles génératives
- `L4` = méta-règles
- `LΩ` = signature minimale fertile

Principe :

```text
bonne compression = moins d'information + plus de pouvoir génératif
```

### 2.2 Tour EXP

```math
EXP(\sigma)=[E_0(\sigma),E_1(\sigma),E_2(\sigma),E_3(\sigma),E_4(\sigma)]
```

- `E0` = instances
- `E1` = familles
- `E2` = hypergraphes
- `E3` = théories
- `E4` = méta-théories

Quasi-adjonction :

```math
LOG(EXP(\sigma)) \approx \sigma
```

```math
X \in EXP(LOG(X))
```

et, en régime créatif :

```math
EXP(LOG(X)) \supseteq \{X\}
```

Les éléments nouveaux sont des mutations fertiles.

## 3. CVCD

```math
CVCD = \text{Calcul Virtuel Compressé par Décompression Contrôlée}
```

Principe :

```text
ne jamais calculer entièrement ce qu'une signature peut décider localement
```

Forme :

```math
CVCD(X)=EXP_\Delta(VIRT(LOG(X)))
```

Couches :

1. calcul compressé sur `σX = LOG(X)` ;
2. calcul virtuel `Yhat = VIRT(σX)` ;
3. décompression contrôlée `YΔ = EXPΔ(Yhat)`.

## 4. Mémoire négative

```math
\mathcal N=\{m: m \text{ est réfuté, stérile, redondant ou trop coûteux}\}
```

Si :

```math
m\in\mathcal N
```

alors :

```text
ne pas explorer m
```

La mémoire négative est un anti-espace de recherche : elle coupe des branches entières.

## 5. Fertilité

```math
\mathcal F(m)=
\frac{U(m)\,G(m)\,V(m)\,C_f(m)}{K(m)+R(m)+1}
```

- `U` = utilité
- `G` = générativité
- `V` = vérifiabilité OAK
- `Cf` = compression fertile
- `K` = coût computationnel
- `R` = risque de bruit / hallucination / confusion

Promotion :

```math
\mathcal F(m)>\tau
```

Mémoire négative :

```math
\mathcal F(m)\le\tau \quad \text{ou} \quad OAK(m)=réfuté
```

## 13. OAK

```math
OAK(m)\in\{O,A,K,S,R,N\}
```

- `O` = observé
- `A` = ancré par définition
- `K` = connu / vérifié
- `S` = spéculatif
- `R` = réfuté
- `N` = non pertinent / bruit

Aucun motif ne devient vérité automatiquement.

## 15. Algorithme complet

```text
Input: X = nombres, textes, théories, documents, observations, prototypes
Initialize H0 = HGFM(X), N0 = inherited negative memory, A0 = fertile archive
For k in 0..K:
  1. LOG: extract signatures, invariants, motifs, contradictions
  2. Negative memory: prune refuted or sterile branches
  3. CVCD: compute virtually on compressed signatures
  4. JKD: choose minimal high-yield actions
  5. YY3: compression, expansion, transmutation
  6. Tensor factorization: decompose signatures into proper factors
  7. HGFM expansion: create nodes, hyperedges, meta-nodes
  8. TGNT: test calculability, limits, memory architecture, fertility
  9. OAK: classify each motif
  10. EXP: generate conjectures, tests, algorithms, visualizations, prototypes, codices
  11. Archive: validated motifs -> A, dead motifs -> N
Output: HK, AK, NK, GK
```
