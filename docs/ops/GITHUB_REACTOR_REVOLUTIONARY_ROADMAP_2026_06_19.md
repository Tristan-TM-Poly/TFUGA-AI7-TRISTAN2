## 3. Five revolutionary but safe upgrades

### Upgrade 1 — Repo Card Standard

Every repo gets a machine-readable card:

```text
name, role, priority, visibility, OAK status, allowed automation, blocked automation, next actions
```

This turns each repository into a typed HGFM node.

### Upgrade 2 — M_MINUS everywhere

Every failed test, ambiguous claim, missing schema, or blocked action becomes a structured anti-pattern.

```text
failure -> repair rule -> future penalty pattern -> safer propagation
```

### Upgrade 3 — Propagation Queue

The system does not blindly push. It generates a queue:

```text
repo, packet type, action, reason, priority, allowed mode
```

The action remains reviewable.

### Upgrade 4 — Canon Registry

Every major claim across math, physics, AI, software and operations should eventually have:

```text
claim, domain, OAK level, evidence, residue, next test, status
```

### Upgrade 5 — Home-lab runner gate

The home PC/server can become a compute node only after:

- branch protection;
- no unreviewed promotion;
- artifact-only scheduled runs;
- clear secrets policy;
- local-only dry runs;
- test logs retained.
