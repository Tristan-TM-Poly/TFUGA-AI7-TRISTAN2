# Cultural Respect Gate — Tristan Systems

Status: B — public respect gate
Date: 2026-07-06

## 0. Purpose

This gate applies when a Tristan document, product, system, brand, agent or company references living cultures, Nations, communities, languages, stories, symbols, territories, protocols or knowledge systems.

Its goal is simple:

```text
respect first, context first, source first, consent when needed, co-creation when appropriate.
```

## 1. Core rule

```text
Do not turn a living culture into decoration.
Do not claim authority that was not given.
Do not use community knowledge as a product feature without context and permission.
```

## 2. Checklist

Before public use, answer:

```yaml
source_known: true_or_false
specific_context_known: true_or_false
use_is_respectful: true_or_false
use_is_not_decorative_only: true_or_false
permission_question_checked: true_or_false
benefit_or_reciprocity_considered: true_or_false
limits_disclosed: true_or_false
human_review_needed: true_or_false
```

## 3. Status levels

| Status | Meaning |
|---|---|
| PASS | Respectful, limited, sourced and safe for public use. |
| REVIEW | Needs more context or human review. |
| BLOCK | Should not be used in current form. |
| CO_CREATE | Requires collaboration or explicit community process. |

## 4. Safe language

Use language like:

```text
I do not speak for these communities.
I use this as a personal symbol of responsibility, listening and respect.
```

Avoid language like:

```text
I represent these communities.
I own this teaching.
This culture is my branding asset.
```

## 5. M-minus seeds

```yaml
- error_type: decorative_culture_use
  countermeasure: remove or add respectful context
- error_type: authority_claim_without_mandate
  countermeasure: replace with personal-responsibility language
- error_type: missing_source_context
  countermeasure: cite context or remove claim
- error_type: product_use_without_permission_review
  countermeasure: require review before productization
```

## 6. Final principle

```text
Respect is not decoration.
Context is not optional.
Consent is not a slogan.
Co-creation is stronger than extraction.
```
