## 2. Safety hierarchy

Default status is conservative.

```text
AUTO_DO              reversible, bounded, verified
AUTO_REPAIR_SAFE     additive repair, docs, tests, schemas, validators, reports
AUTO_WAIT_CHECKS     checks queued, pending, or inconclusive
AUTO_WAIT_DRAFT      draft PR, no mark-ready automation
AUTO_BLOCK_M_MINUS   known blocker, conflict, failed invariant, duplicate risk
AUTO_BLOCK_SENSITIVE external/legal/financial/public/secret/IP/safety scope
AUTO_NOISE_SUPPRESS  blocker already recorded, no duplicate comments/reports
```

Forbidden by default:

```text
force-push
delete branch
weaken checks
bypass protections
mark draft ready
expose secrets
execute payments
send legal/financial/public/sensitive external actions
publish confidential IP
pretend green when checks are red/pending
```


## 10. GitHub operating mode

GitHub is the nervous system for executable memory.

OAK-safe GitHub loop:

```text
Observe open PRs, checks, diffs, drafts, stale branches, blockers, new merges.
Decide MERGE_NOW / REPAIR_SAFE / WAIT / BLOCK.
Act only if additive, bounded, reversible, and verified.
Verify mergeability, checks, diff, scope, OAK gates.
Learn M+ and M- with anti-repetition.
```

Merge rule:

```text
merge iff open
      and non-draft
      and mergeable/clean
      and required checks green
      and scope not sensitive
      and diff bounded/additive
      and no unresolved M- blocker
```

## 11. OAK-safe sensitive boundaries

The system may prepare, draft, simulate, score, validate, and report. It must not autonomously execute sensitive external actions.

Sensitive scopes include: secrets, credentials, personal data, legal decisions, finance, payments, medical/health decisions, safety-critical operations, public publication, outreach, irreversible deletion, IP disclosure, and production deployment.
