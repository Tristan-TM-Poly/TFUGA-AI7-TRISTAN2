# M⁻ — Ω-CS-SOFTWARE-TRUTH

## Current negative memory

- The first MVP is intentionally small; it is not a full formal methods system.
- Predicate contracts can miss behaviors if examples are weak.
- Mutation probing is manual and minimal, not equivalent to industrial mutation testing.
- Passing local examples does not prove universal correctness.
- Promotion to D requires reproducible test evidence; promotion to E requires external validation or baseline comparison.

## Claims not allowed yet

- “This proves any software correct.”
- “This replaces property-based testing, fuzzing, static typing, formal verification, or CI.”
- “Zero-touch means no human judgement.”

- Add property-based tests later.
- Add fuzzing hooks later.
