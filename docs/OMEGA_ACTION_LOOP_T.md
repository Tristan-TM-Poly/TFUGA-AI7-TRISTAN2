# Ω-ACTION-LOOP-T — Boucles d’action intelligentes de Tristan

## Mission

Ω-ACTION-LOOP-T formalise les boucles d’action continues mais intelligentes du système Tristan : GitHub, universités, Gmail, packets de preuve, microbenchmarks, OAK, M+/M−, revenus et actifs.

Le but n’est pas d’agir sans arrêt aveuglément. Le but est d’agir en cycles courts, sûrs, vérifiables et cumulatifs.

```text
Observe → Decide → Act → Verify → Learn → Compress → Reprioritize → Repeat
```

Version Tristan :

```text
Voir → Juger → Agir → Vérifier → Apprendre → Compresser → Reboucler
```

## Principe central

```text
Une action n’est bonne que si elle augmente le capital de preuve.
```

Chaque boucle doit produire au moins une sortie utile : preuve, code, doc, test, route officielle, packet, PR fusionnée, réponse, rapport OAK, M− utile ou prochaine action plus facile.

## Lanes canoniques

| Lane | Rôle |
|---|---|
| GitHub Action Loop | scanner, réparer, fusionner seulement si clean/OAK-safe |
| University Action Loop | greenifier les routes officielles, préparer packets, éviter spam |
| Reply Action Loop | transformer réponses Gmail en meeting cards, packets ou M− |
| Demo Packet Loop | produire preuves concrètes réutilisables |
| OAK/M− Loop | empêcher les surpromesses et erreurs répétées |
| Value Loop | transformer preuves et relations en actifs, IP, revenus, publications |

## Décisions possibles

```text
MERGE_NOW
REPAIR_SAFE
PREPARE_PACKET
WAIT_COOLDOWN
BLOCK_M_MINUS
HUMAN_APPROVAL_REQUIRED
```

## Garde-fous

- Pas de force-push.
- Pas de suppression de branches sans approbation explicite.
- Pas de secrets exposés.
- Pas de bypass de protections.
- Pas de merge de PR draft.
- Pas de merge avec checks pending/ambigus.
- Pas d’adresse email devinée.
- Pas de mass-send.
- Pas de claim industriel/ROI/partenariat sans preuve.
- Pas d’action légale, financière, publique, santé, sécurité ou IP sensible sans approbation.

## Cadences intelligentes

```text
GitHub checks pending → attendre et réessayer plus tard
Universités sans réponse → relance unique après 7–10 jours
Routes officielles non trouvées → recherche hebdomadaire
PR draft → ne jamais fusionner tant que draft
Formulaire officiel → packet prêt, soumission seulement avec confirmation
```

## Boucle finale

```text
GitHub scan
→ PR safe merge/repair
→ docs/tests/packets créés
→ université scan
→ route verte trouvée
→ message court
→ réponse
→ packet adapté
→ microbenchmark
→ OAK report
→ preuve externe
→ publication/brevet/revenu
→ M+/M−
→ boucle suivante
```

## Objectif profond

Ω-ACTION-LOOP-T transforme la friction en matière première : chaque blocage devient soit une action sûre, soit un packet, soit un M−, soit une prochaine meilleure route.
