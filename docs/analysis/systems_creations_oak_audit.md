Ce document transforme l'analyse du corpus en carte actionnable. Il ne certifie pas les claims physiques : il sépare les niveaux **vision**, **définition**, **formalisme**, **prototype**, **simulation**, **mesure**, **preuve**, **canon** et **M_MINUS**.

## 1. Résumé exécutif

Le corpus décrit un écosystème de recherche-création centré sur :

```text
TFUGA / TGTM / HGFM / CVCD / OAK / SAGE / AIT
```

Sa valeur principale n'est pas encore dans des preuves universelles certifiées, mais dans une architecture générative déjà riche, partiellement traduite en prototypes logiciels, avec un système immunitaire épistémique explicite : OAK + mémoire négative.

Noyau sain :

```text
intuition générative
-> compression / canonisation
-> prototype minimal
-> attaque OAK
-> mémoire négative
-> version vérifiable
```

Faille principale : le style Go Max / Max Plus Ultra peut amplifier les possibilités avant que les statuts de preuve, mesure, simulation et spéculation soient séparés. Le remède n'est pas de réduire l'ambition, mais de la canaliser dans un protocole :

```text
Document -> Calcul -> Test -> Résidu -> M_MINUS / CANON
```

## 6. Priorités parallèles recommandées

### Piste A — Corpus / Canon

Créer un **Tristan Canon Registry** :

```text
canon/tristan_canon_registry.seed.yml
```

Chaque concept doit porter : définition, statut, sources, dépendances, tests, risques, prochaine action.

### Piste B — Spectroscopie / physique mesurable

Objectif : 50–500 spectres Raman/FTIR/XRD publics.

Mesures : reconstruction, classification, SNR, RMSE, R2, MAE, F1, stabilité de pics, mémoire négative.

### Piste C — Échecs comme banc d'essai OAK

Objectif : transformer AIT-ChessMaster en laboratoire de validation :

- perft ;
- tablebases sur finales simples ;
- Stockfish comme oracle ;
- divergence -> M_MINUS ;
- stabilité score/profondeur.

### Piste D — LC/RLC passif

Ne pas commencer par claims d'énergie. Commencer par :

- circuit LC fractal passif ;
- simulation SPICE ;
- résonances ;
- facteurs Q ;
- pertes ;
- comparaison modèle/mesure ;
- aucun claim au-delà des données.

### Piste E — Zero-touch local

Créer un script unique :

```text
scripts/analyze_all.py
```

Fonctions minimales : indexer documents, extraire claims, assigner statuts OAK initiaux, générer rapport, générer backlog, conserver `m_minus.jsonl`.
