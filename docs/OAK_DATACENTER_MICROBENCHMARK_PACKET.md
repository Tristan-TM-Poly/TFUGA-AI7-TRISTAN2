# OAK Datacenter Microbenchmark Packet

## Mission

Create a concrete proof packet for university outreach and research collaboration around energy, optimization, datacenters, ROI-OAK, and scientific AI.

## Why this packet exists

The Quebec university outreach engine needs proof artifacts. A generic message can open a door, but a specific OAK-safe packet can turn interest into a concrete microbenchmark.

This packet makes the proposal reviewable:

```text
short explanation → example report → code references → university fit → safe email variant
```

## Existing software basis

The packet references the existing MVP:

```text
omega_vtp_t/datacenter_thermal.py
examples/datacenter_oak_demo.py
```

The MVP estimates a conservative thermal/energy surrogate report with:

- PUE proxy;
- cooling reduction scenario;
- hotspot risk score;
- residual checks;
- invariant checks;
- finance/ROI-OAK decision;
- unified OAK decision.

## OAK interpretation

This packet is valid for:

- academic discussion;
- microbenchmark proposal;
- student project framing;
- early validation design;
- prototype demonstration;
- controlled pilot scoping.

It is **not** valid for:

- claiming industrial savings;
- replacing CFD;
- bypassing facility engineering;
- claiming deployment readiness;
- claiming university endorsement.
