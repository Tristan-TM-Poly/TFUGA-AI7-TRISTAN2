# OAK Report — Ω-AIT-REALITY-FORGE-T

## Scope

This report documents the OAK boundaries for the Reality Forge layer.

Reality Forge converts ideas into structured, labeled, tested, reversible artifacts. It does not authorize high-stakes action, medical/legal/financial/security decisions, public release, or irreversible automation.

## OAK status

- **Observe:** classify RealityLevel R0-R10 and ProofLevel P0-P10.
- **Act safely:** compile only reversible artifact plans by default.
- **Kill-risk:** quarantine boundary-violating, public, irreversible, destructive, medical, legal, financial, security, privacy, or IP-sensitive ideas.

## Core safety rules

1. No vision-to-proof jump.
2. No R10 language for R2 evidence.
3. No ZERO-TOUCH for irreversible actions.
4. No public release without privacy/IP review.
5. No medical/legal/financial/security replacement.
6. Every strong claim needs RealityAnchor, counterworlds, failure modes, proof level, and artifact plan.

## Required artifacts

- theory doc;
- schema;
- tool;
- tests;
- negative memory;
- OAK report.

## Merge policy

This layer should remain draft until reviewers check:

- wording avoids overclaiming;
- tests match conservative semantics;
- R/P levels are not presented as proof;
- high-stakes domains downgrade autonomy;
- artifact compiler does not execute external actions;
- repository organization is acceptable.
