# OAK Report — Ω-AIT-NO-HUMAN-BOTTLENECK-T

## Scope

This layer removes routine human bottlenecks. It requires the AIT to continue by creating the strongest safe artifact available: files, tests, simulations, reports, draft PRs, review packets, quarantine notes, M− entries, or next-action plans.

It does not remove human sovereignty at OAK thresholds.

## OAK status

- **Observe:** identify whether the task is reversible, private, testable, public-facing, review-scoped, or irreversible.
- **Act safely:** continue zero-touch for reversible/private/testable work.
- **Kill-risk:** downgrade to simulation, review packet, OAK report, or hold when a threshold is reached.

## Core boundaries

1. No routine human bottleneck for safe reversible work.
2. No bypass of review thresholds.
3. No direct external action when simulation/report/draft PR is the safer next artifact.
4. No auto-merge and no auto-publish.
5. Review threshold is not a stop; it is a routing event.
6. Even when direct execution is held, produce the best safe next artifact.

## Review checklist

- ContinuationModeRouter should route safe work to create mode.
- Public-facing or commitment-like work should route to review mode.
- Irreversible or qualified-review domains should route to hold mode.
- DowngradePacket should list missing evidence, gates, tests, and next safe action.

## Final law

```math
\boxed{\text{Never stop. Downgrade autonomy instead.}}
```
