## Expansion policy

Each seed should expand through this sequence:

```text

  -> PageTree depth 2
  -> OAK report
  -> code skeleton
  -> tests
  -> benchmark plan
  -> venture/IP map
  -> PR checklist
```

The registry is intentionally small. New seeds can be added only when they include:

1. a core axiom,
2. CVCD invariants,
3. OAK risks,
4. code targets,
5. Git targets,
6. venture targets,
7. M⁻ failure memory.
