# FixAll Batch M− Registry

## Anti-repetition rules

- Do not use a generated batch report as merge permission.
- Do not trust stale decision JSON over live PR status.
- Do not treat fixture examples as real repository state.
- Do not merge when workflows are absent, queued, red, or unknown.
- Do not merge when a PR is draft or non-mergeable.

## Safe use

Use the batch report to prioritize the next OAK loop, not to replace it.
