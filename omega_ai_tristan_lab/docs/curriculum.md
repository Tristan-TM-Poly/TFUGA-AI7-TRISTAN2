# Curriculum Ω-AI-TRISTAN-LAB

Objectif : remplacer l'apprentissage passif par une machine à produire des artefacts vérifiables.

## Principe directeur

```text
Ne jamais apprendre sans produire.
```

Chaque module doit sortir au moins un fichier, un test, un rapport OAK ou une décision IP/revenu.

## Modules

| Module | Compétence | Artefact |
|---|---|---|
| 1. Fondations IA | Comprendre ML, LLM, embeddings, RAG, agents | `ai_foundations_tristan.md` |
| 2. RAG personnel | Transformer corpus en contexte cité | `rag_engine.py` |
| 3. Agentic AI | Planifier, agir, vérifier | `agent_harness.py` |
| 4. OAK | Réduire hallucinations et surpromesses | `oak_eval.py` |
| 5. Bayes-Tristan | Décider sous incertitude multi-axe | `bayes_tristan.py` |
| 6. IP gate | Protéger inventions et secrets | `ip_classifier.py` |
| 7. Revenue map | Transformer prototypes en hypothèses économiques | `revenue_mapper.py` |
| 8. GitHub factory | Branches, PR, tests, docs | PR reproductible |
| 9. Paper-to-prototype | PDF → LaTeX/code/tests | pipeline Rosette-Tristan futur |
| 10. Capstone | Idée → prototype → OAK → IP/revenu | portfolio publiable/privé |

## Niveaux de maîtrise

| Niveau | Preuve |
|---|---|
| 1 | Lire et expliquer un concept IA |
| 2 | Construire un mini-outil |
| 3 | Tester l'outil |
| 4 | Comparer à une baseline |
| 5 | Cartographier IP/revenu |
| 6 | Livrer un prototype GitHub |
| 7 | Obtenir un signal utilisateur/client |
| 8 | Produire publication, brevet, service ou SaaS |

## Capstone

Construire un agent qui prend une idée brute et sort :

1. une TheoryCard,
2. un OAKReport,
3. un BayesAxisScore,
4. une IPClassification,
5. une RevenueMap,
6. une prochaine action minimale.

## Règle OAK

Aucune revendication forte sans : source, test, baseline, incertitude, limite et résidu.
